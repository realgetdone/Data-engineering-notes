The docker process

→ 1) We write the docker file

2) We then build the docker images

3) We then run a container from the docker image

Detailed process

  1. Write a Dockerfile:
    • Purpose: Define the setup and configuration for your application environment.
    • Contents: The Dockerfile contains a set of instructions that specify how the Docker image should be built. These instructions might include choosing a base image, copying application files, installing dependencies, and defining the command to run the application.

    Example Dockerfile:

    FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]
  1. Build the Docker Image:
    • Command: docker build -t my-node-app .
    • Purpose: Create a Docker image from the Dockerfile. This process reads the Dockerfile, executes the instructions in order, and assembles the image.
    • Result: A Docker image named my-node-app is created. This image contains all the necessary files and configurations as specified in the Dockerfile.
  1. Use the Docker Image:
    • Command: docker run -p 3000:3000 my-node-app
    • Purpose: Run a container from the my-node-app image. The container is a running instance of the image, with its own isolated environment.
    • Result: A Docker container is created and started based on the image. It runs the application as specified in the Dockerfile.

Summary

  • Dockerfile: A script with instructions to build a Docker image.
  • Docker Image: Created from the Dockerfile, it is a snapshot of the application environment.
  • Docker Container: A running instance of the Docker image, providing an isolated environment for your application.

In essence, you write a Dockerfile to define your application's environment and configuration, and then use that Dockerfile to build a Docker image. This image can be used to create and run containers with a consistent setup across different environments.

Build the docker image 

docker build -t my-python-image .

  • my-python-image: This is the name (or tag) you are assigning to the Docker image you are building. You can choose any name you like.

  • .: This refers to the current directory. It specifies the build context for Docker. Docker will look for the Dockerfile and any files needed for the build in this directory.

  • docker build: The Docker command to build an image from a Dockerfile.

  • t my-python-image: The t flag tags the image with the name my-python-image. You can also use a format like my-python-image:latest to specify a tag.

  • .: The dot indicates that the build context is the current directory. Docker will look for a Dockerfile in this directory and use it to build the image.
